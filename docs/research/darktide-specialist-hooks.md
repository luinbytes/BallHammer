# Darktide specialist hook research

Research snapshot: 2026-07-17. Darktide script mirror commit [`47379fd`](https://github.com/Aussiemon/Darktide-Source-Code/tree/47379fd3cbb6d59c3e9001bab1693c307bf46e2b), which identifies itself as decompiled game scripts for patch 1.11.7, and DMF commit [`b9cc65f`](https://github.com/Darktide-Mod-Framework/Darktide-Mod-Framework/tree/b9cc65f773cd8aaa974bf5b9312a79f5c5785f90). Fatshark does not publish a supported gameplay mod API, so game-internal symbols below are evidence from the current public script mirror, not a compatibility guarantee.

## Feasibility summary

| Module | Static evidence | Runtime prototype still needed |
| --- | --- | --- |
| Threat Interceptor | Strong for identifying attacker, target, and running behavior | Yes. Precise commitment timing and direction, especially generic elite overheads |
| Guard Brain | Strong. BallHammer already injects networked raw input at the correct seam | Yes. Switch-to-melee then block latency and action validity across all weapons |
| Warp and Heat Governor | Strong for current resource, stopping input, and invoking each weapon's native vent action | Yes. Exact future peril/heat cost after talents, buffs, and charge state |
| Armor and Weakspot Director | Strongest. Native breed, hit-zone, weakspot, armor, and damage-profile data exist | Yes, but mostly for target-point visibility and representative weapon coverage |

## 1. Detecting committed specialist attacks

The common read path exists. [`AiBrain.running_action`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/ai_brain.lua#L268-L280) returns the active leaf identifier and its `action_data`; [`MinionBehaviorExtension.running_action`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/minion_behavior_extension.lua#L241-L243) exposes it through `ScriptUnit.has_extension(unit, "behavior_system")`. The game itself uses this route in [`Dodge.sucessful_dodge`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/character_state_machine/character_states/utilities/dodge.lua#L217-L219). `BLACKBOARDS[unit].perception.target_unit` is the current target used by the actions.

Concrete running leaf names and action classes are:

- Dogs: `leap` in the [hound tree](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/trees/chaos/chaos_hound_behavior_tree.lua#L60-L77). The shared blackboard `pounce.started_leap` is set at the actual launch in [`BtChaosHoundLeapAction._leap`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_chaos_hound_leap_action.lua#L342-L353), making this the cleanest committed signal.
- Trappers: `shoot_net` in the [netgunner tree](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/trees/renegade/renegade_netgunner_behavior_tree.lua). [`BtShootNetAction`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_shoot_net_action.lua#L143-L226) tracks `shoot_t` and moves from `aiming` to `shooting`; the phase is scratchpad-only.
- Mutants: `charge` in the [mutant tree](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/trees/cultist/cultist_mutant_behavior_tree.lua#L52-L61). [`BtMutantChargerChargeAction._start_charging`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_mutant_charger_charge_action.lua#L174-L194) is a concrete commitment hook; collision timing remains action scratchpad data.
- Snipers: `shoot` in the [sniper tree](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/trees/renegade/renegade_sniper_behavior_tree.lua#L20-L30). [`BtSniperShootAction`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_sniper_shoot_action.lua#L162-L239) exposes distinct start/update aiming methods, then [`_start_shooting`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_sniper_shoot_action.lua#L438-L459).
- Flamers: `shoot` backed by `BtShootLiquidBeamAction` in the [flamer tree](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/trees/renegade/renegade_flamer_behavior_tree.lua#L18-L32). Its [`_start_shooting`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_shoot_liquid_beam_action.lua#L339-L367) is concrete, but a one-shot dodge is not necessarily the correct response to a sustained area beam.
- Grenadiers: `throw_grenade` and `quick_throw_grenade` in the [grenadier tree](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/trees/renegade/renegade_grenadier_behavior_tree.lua#L13-L40). [`BtGrenadierThrowAction._start_aiming`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_grenadier_throw_action.lua#L81-L110) derives the exact throw time, and [`_throw_grenade`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_grenadier_throw_action.lua#L135-L179) is the projectile-created boundary.
- Elite overheads: the common leaf is only `melee_attack`, for example in the [Ogryn executor tree](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/trees/chaos/chaos_ogryn_executor_behavior_tree.lua#L9-L49). [`BtMeleeAttackAction._start_attack_anim`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/behavior/nodes/actions/bt_melee_attack_action.lua#L120-L199) chooses an animation and damage timing from `action_data`; the chosen animation and next hit time live in scratchpad. There is no universal `overhead` flag.

Recommended minimum: poll `running_action()` plus target identity, use `pounce.started_leap` for dogs, and hook the action-class commitment methods for the remaining specialists. Do not infer exact timing from distance alone. Unknown action or missing target should produce marker-only behavior.

## 2. Dodge, block, push, and emergency melee switch

The networked input list explicitly contains movement axes, `action_two_hold`, `action_two_pressed`, `dodge`, and `wield_1` in [`input_handler_settings.lua`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/managers/player/player_game_states/input_handler_settings.lua#L3-L48). [`HumanInputHandler._parse_input`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/managers/player/player_game_states/human_input_handler.lua#L232-L270) writes the fixed-frame input cache. BallHammer already hooks this exact method, so defensive inputs should share that seam rather than introduce a second simulator.

- Dodge: set the ephemeral `dodge` cache entry for one fixed frame. A directional dodge also depends on movement-axis values; [`Dodge.check`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/character_state_machine/character_states/utilities/dodge.lua#L23-L75) reads both the dodge input and movement/settings. Preserve the unmodified cache first so physical input can suppress automation.
- Block: melee weapon templates route `action_two_hold` through their action input hierarchy into [`ActionBlock`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/weapon/actions/action_block.lua). The meaning of right-click is weapon-specific, so validate the current template action input instead of assuming every equipped item can block.
- Push: while blocking, melee templates normally map `action_one_pressed` to [`ActionPush`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/weapon/actions/action_push.lua). The action itself checks stamina and degrades to a weak push when insufficient, so the guard should still check stamina and threat geometry before opting in.
- Emergency melee switch: [`PlayerCharacterConstants`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/settings/player_character/player_character_constants.lua#L617-L640) maps `slot_primary` to `wield_1` and `slot_secondary` to `wield_2`. Inject `wield_1`, then wait until the inventory component reports `wielded_slot == "slot_primary"` and the block action is valid. The direct utility [`PlayerUnitVisualLoadout.wield_slot`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/visual_loadout/utilities/player_unit_visual_loadout.lua#L12-L34) exists, but raw network input is safer for prediction consistency.

DMF supports both post-call callbacks and wrapped replacement hooks through [`DMFMod:hook_safe` and `DMFMod:hook`](https://github.com/Darktide-Mod-Framework/Darktide-Mod-Framework/blob/b9cc65f773cd8aaa974bf5b9312a79f5c5785f90/dmf/scripts/mods/dmf/modules/core/hooks.lua#L426-L440). BallHammer's existing wrapped input hook is therefore the right implementation surface.

## 3. Psyker peril and weapon heat

Peril is directly readable from the player unit-data component `warp_charge.current_percentage`; this is the same field read by the game's [`ActionVentWarpCharge`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/weapon/actions/action_vent_warp_charge.lua#L14-L48). Heat is stored per inventory slot as `overheat_current_percentage`, read by [`OverheatActionModule`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/weapon/actions/modules/overheat_action_module.lua#L14-L46). Read `inventory.wielded_slot`, then that slot component; do not assume heat is on the player-wide warp component.

Native action behavior already supplies conservative stop boundaries. [`WarpChargeActionModule.running_action_state`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/weapon/actions/modules/warp_charge_action_module.lua#L50-L62) reports fully charged at peril `>= 1`; [`OverheatActionModule.running_action_state`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/weapon/actions/modules/overheat_action_module.lua#L32-L56) stops increasing heat at `>= 0.99`. A governor can release or suppress generated fire before a configured lower threshold, but exact next-shot cost is affected by charge templates and buffs and must be measured in a prototype.

Quell/vent should be invoked through the current weapon template's `action_inputs.vent`, not a universal hard-coded key. For example, a [force staff maps `vent` to `weapon_reload_hold`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/settings/equipment/weapon_templates/force_staffs/forcestaff_p3_m1.lua#L101-L120), while [plasma maps `vent` to `weapon_extra_hold`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/settings/equipment/weapon_templates/plasma_rifles/plasmagun_p1_m1.lua#L89-L110). The actions are [`ActionVentWarpCharge`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/weapon/actions/action_vent_warp_charge.lua) and [`ActionVentOverheat`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/weapon/actions/action_vent_overheat.lua); plasma venting can damage the player above its low threshold, so safe-to-vent must include current health and nearby threat checks.

## 4. Weakspots, armor, and weapon capability

The target breed is available from `ScriptUnit.has_extension(unit, "unit_data_system"):breed()`, implemented by [`MinionUnitDataExtension.breed`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/extension_systems/unit_data/minion_unit_data_extension.lua#L110-L112). Breed tables contain `hit_zones`, `hit_zone_weakspot_types`, base `armor_type`, and optional `hitzone_armor_override`; the [sniper breed](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/settings/breed/breeds/renegade/renegade_sniper_breed.lua#L182-L266) and its [head weakspot mapping](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/settings/breed/breeds/renegade/renegade_sniper_breed.lua#L410-L412) are representative.

Use the game utilities rather than copying those tables:

- [`Weakspot.hit_weakspot`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/utilities/attack/weakspot.lua#L9-L19) accounts for breed weakspot types and guaranteed-weakspot buffs.
- [`Armor.armor_type`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/utilities/attack/armor.lua#L11-L66) accounts for hit-zone overrides, toughness armor, Havoc rotten armor, and buff overrides. Reading only `breed.armor_type` would be wrong in those states.
- [`HitZone.get_actor_names`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/utilities/attack/hit_zone.lua#L31-L38) and [`HitZone.hit_zone_center_of_mass`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/utilities/attack/hit_zone.lua#L96-L119) provide real target geometry instead of BallHammer's current height-fraction approximation.

Weapon capability is action-specific, not a single weapon flag. [`WeaponTemplate.current_weapon_template`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/utilities/weapon/weapon_template.lua) resolves the equipped template from the `weapon_action` component. The relevant firing or sweep action then points to a damage profile whose `armor_damage_modifier` or `armor_damage_modifier_ranged` encodes performance by armor type; the game feeds those modifiers into [`DamageCalculation.calculate`](https://github.com/Aussiemon/Darktide-Source-Code/blob/47379fd3cbb6d59c3e9001bab1693c307bf46e2b/scripts/utilities/attack/damage_calculation.lua). A minimal director should rank visible hit zones using the current action's actual damage profile and `Armor.armor_type`, and fail closed when the active action has no inspectable damage profile.

## Prototype gates before enabling automation

1. Log running action, target, blackboard phase, and hook timestamp for each specialist in the Psykhanium or a private mission. Confirm the client sees the commitment early enough to act.
2. Exercise injected `dodge`, switch-to-melee, block, and push through the fixed-frame network cache with rollback/mispredict handling. Confirm physical input always suppresses them unless emergency override is enabled.
3. Sample peril and heat before and after representative actions across Psyker staves, force swords, plasma, and overheat power weapons. Keep configurable margins rather than claim exact universal prediction.
4. Compare selected actor-backed aim points against real weakspot and armor results for unarmored, flak, carapace, shielded, and dynamic Havoc armor targets.

The safe first implementation boundary is armor/weakspot ranking plus marker-only threat detection. Automatic reactions should remain opt-in until the four runtime gates above pass.
